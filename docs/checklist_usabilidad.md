# Checklist de usabilidad

Usar antes de Entrega 3 y antes de abrir nuevas ramas funcionales.

## Navegación

- [ ] ¿El dashboard se entiende sin explicación?
- [ ] ¿Nueva orden de carga está visible como acción principal?
- [ ] ¿El menú es claro y está organizado por secciones?
- [ ] ¿La opción activa se distingue del resto?
- [ ] ¿Los módulos futuros no parecen rotos?

## Pantallas de trabajo

- [ ] ¿Las tablas se leen bien?
- [ ] ¿Los filtros o buscadores están visibles?
- [ ] ¿Los estados activo/inactivo o disponible/ocupado se entienden?
- [ ] ¿La pantalla de orden de carga se puede usar sin capacitación extensa?
- [ ] ¿Los botones principales tienen nombres concretos y consistentes?

## Mensajes

- [ ] ¿Los mensajes son amigables?
- [ ] ¿No aparecen errores Python, rutas internas ni nombres de clases?
- [ ] ¿Los módulos pendientes muestran "Funcionalidad prevista para una próxima entrega."?
- [ ] ¿Las validaciones explican qué debe corregir el usuario?

## Impresión y entrega

- [ ] ¿La impresión A4 se ve aceptable?
- [ ] ¿Incluye empresa, título, número, fecha, cliente, despacho, transporte, totales y firma?
- [ ] ¿Se puede mostrar al cliente?
- [ ] ¿Hay capturas actualizadas en `docs/screenshots/ux_base_final/`?
- [ ] ¿Los tests y smoke checks fueron ejecutados?
